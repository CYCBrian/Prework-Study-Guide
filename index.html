HTML

  • Hypertext Markup Language (HTML) - to give the web content structure (i.e. the frame).

  • Element - a type of content on a webpage, used to add content to the pages themselves, to divide the content on the page, and to describe the content to browswers and other developers.

  • Attribute - defines additional information about an element. NOTE: attributes are always defines in the beginning tag, never the closing tag.

Git

  • git status - to check which brach we are currently on.

  • git clone "SSH Code" - clones the repository from the clous to your local computer.

  • git more -v - to ensure that there is a connection ebtween the local repository and remote repository.

  • git checkout main - to move (chekcout) from the current working branch to the main brach (main)

  • git checkout -b "Branch Name" - to move (chekcout) from the current working branch to a new branch (-b) called "Branch Name"

  • git add -A - to add modifications in the current working branch to the stagin area. The -A flag indicates that we want to add all changes in the current working branch.

  • git commit -m "add ..." - a commit message that describes what the commit contains. Creates a snapshot of the currently stage changes.

  • git pull origin main - to receive (pull) a branch's modification into the local environment. (origin) indicates te source of the pull will be in the GitHub repo. (main) indicates the branch. In this case, the main branch.

  • git push origin "Branch Name" - to push (push) the changes we have locally to our remote GitHub branch.
