HTML

  • Hypertext Markup Language (HTML) - to give the web content structure (i.e. the frame).

  • Nesting - when one or more elements are placed inside another elements, allow applying of multiple HTML tags to a single piece of content. The entire element (including its markup) must be within the start and end tags of the containing element (the parent).

  • Child elements - elements that are nested inside other elements.

  • Parent element - the element that nests the child elements.

  • Element - a type of content on a webpage, used to add content to the pages themselves, to divide the content on the page, and to describe the content to browswers and other developers.

  • Metadata - elements that contain data about the webpage that the browser uses to serve the webpage to users. Not seen by users, but contains information about accessibility, search engines, and performance.

  • Semantic HTML elements - the name of the element describing its content and the role it plays on a webpage. Makes it easier to read for developers and browsers. Ensures accessibility to a variety of uers (e.g. those who use screen readers). Not mandatory, but best practise is to use it where possible.

  • head - element containing metadata that helps the browser to read the page correctly. Not visible when a user looks at a webpage.

  • body - element containing the content that is shown to the user. All the elements that we want the user to see and interact with must be placed inside this element.

  • !DOCTYPE "doct ype" - element of declaration to tell the browser what document type it should expect when it is opened.

  • title - element that defines the title for a webpage. Required for Search Engine optimization (SEO).

  • style - element that defines the CSS styles associated with a website.

  • base - element that defines the base URL for a webpage.

  • link - element that connects an external resource to the HTML document.

  • script - element that embeds the client-side scripts in the HTML document.

  • meta charset="charset" - element that defines the characters set (e.g. UTF-8).

  • meta http-equiv="http-equiv" - element that sets the HTTP header for content.

  • meta viewpoint-"viewpoint" - element that sets the default view for our webpage (i.e. the first thing you see e.g. homepage).

  • div - element that creates a division in the page.

  • span - element that creates an inline division.

  • h"n" - element to represent the level of heading a given text block represents. h1 is the largest, then gets smaller and smaller. h1, the biggest header, should only appear once on each page.

  • p - element to represent a paragraph or block of text.

  • ul - element to represent an unordered list.

  • ol - element to represent ordered list.

  • li - element to represent an list item.

  • img - element that contains information about images that are displayed on the webpage.

  • br - element that creates a line of empty space, or a line break, between two blocks of content.

  • a - element that creates links to the same webpage or other webpages.

  • selection - semantic element that defines a section in a document that contains thematic content.

  • Attribute - defines additional information about an element. NOTE: attributes are always defines in the opening tag, never the closing tag.

  • src="" - attribute that defines the location of an image. Placed inside of the img element.

  • alt="" - attribute that defines alernative information for an image if a user cannot view it. Important for assistive technologies, and can improve search engine rnkings for your website.

  • href="" - attribute that points to the URL for the link.

  • id="" - attribute to identify elements and select them.

Git

  • Git commands - alwyas start with git and are used to save and track versions of your code. Using Git commands, we can initiate (or create) a repo, add our code to the repo, and view the different versions of the ocode, which are each stored in a unique commit.

  • git status - to check which branch we are currently on.

  • git clone "SSH Code" - clones the repository from the cloud to your local computer.

  • git remote -v - to ensure that there is a connection between the local repository and remote repository.

  • git checkout main - to move (chekcout) from the current working branch to the main brach (main).

  • git checkout -b "Branch Name" - to move (chekcout) from the current working branch to a new branch (-b) called "Branch Name".

  • git add -A - to add modifications in the current working branch to the staging area. The -A flag indicates that we want to add all changes in the current working branch.

  • git commit -m "add ..." - a commit message that describes what the commit contains. Creates a snapshot of the currently stage changes.

  • git pull origin main - to receive (pull) a branch's modification into the local environment. (origin) indicates the source of the pull will be in the GitHub repo. (main) indicates the branch. In this case, the main branch.

  • git push origin "Branch Name" - to push (push) the changes we have locally to our remote GitHub branch.

  • git branch - displays a list of all available branches.

Termanologies

  • SSH key - an access credential for the secure shell (SSH) network protocal. Generating SSH keys allow developers to interface with certain remote services without having to constantly retype login information.

  • MVP - a minimum viable product.

  • SHA (hash) - to access a version of the code.

  • Git - a version control system that lets you manage and keep track of each version of your project. A tool that allows developers and others to use version control.

  • GitHub - one of many web interfaces for using Git.

  • Organization (org) - a grouping mechanism allowing teams to collaborate across many projects at once.

  • Repository (repo) - a folder in which all files and their version histories are stored.

  • Branch - a version of the repo that allows work without affecting other branches. Repos may have many branches for different possible changes being tested or considered, along with a default branch that serves as the main version of code that all other branches are merged into.

  • Fork - a new repository that inherits from a parent "upstream" repo. It is used to suggest changes to the "upstream" public repo by someone who doesn't have access to edit in the repo's home org.

  • Markdown (.md) - a way to write content that converts plan text to formatted text. A plain text formatting syntax aimed at making writing for the internet easier.

  • Commit Chnages - a saved record of the changes made to a file within the repo.

  • Pull Request (PR) - a request for changes made to a branch to be pulled into another branch. Allows multiple users to see, discuss and review working being suggested.

  • Merge - after a pull request is approves, the commit will be pulled in (or merged) from one branch to another and then, deployed to the live site.

  • Issues - allow users to report issues/bugs and track progress of assigning the fix for the issues.

  • Federalist - a platform that securely deploys a website from a GitHub repository and lets users preview proposed and published changes.

  • Projects - allows you to use GitHub for project management and tracking a set of issues, either for a specific repo or an entire org.

  • Wiki - a section of a repo made for hosting documentation. Documentation may be in the repo's README files instead.

  • Readme file - uses the markdown language with extention .md. Can contain descriptions of a repository, images, techs used, and set up instructions.

  • .gitignore file - tells git to not trach this file from our local machine. Useful as a security measure. To use, type the name of the file that contain your personal information inside of the .gitignore file, so that that file is ignored when syncing the local code with the repository in the cloud.

  • Fetch - fetch any new changes from the remote repository to the local repository.

  • Push - push any code from the local repository to the remote repository.

  • GitHub issues - give devlopers and teams an easy way to keep track of key tasks and progress. Track, organize, and communicate ideas, tasks, bugs, and enchancements. Identify bottlenecks or blockers that can prevent work progress. Avoid duplicating work by sassigning tasks or issues.

  • Blocker - getting stuck on a particular problem that is preventing you from moving forward.

  • Bottleneck - a point in the development process that is slowing down the rest of production.

  • Feature - a functional part of the application or website that has value or fulfills a need in the business.

  • Intergrated Development Environment (IDE) - a type of software used by developers that combines many of the features needed by developers to do their work in a single interface, known as a graphical user interface (GUI).

  • Uniform Resource Locator (URL) - unique address given to a specific resource on the web.
